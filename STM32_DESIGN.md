## 2. Module Specifications

### A. Traffic Density Monitor (`traffic_monitor.c`)
*   **Purpose:** Reads raw vehicle count files generated by the scenario generator and outputs dynamic traffic profiles for each junction.
*   **Inputs:** `simulation/scenarios/traffic_scenario.json` (simulation mode) or virtual sensor registers.
*   **Outputs:** Global arrays containing vehicle count integers and categorized density classifications:
    ```c
    typedef enum { LOW_DENSITY, MED_DENSITY, HIGH_DENSITY } DensityClass;
    uint8_t junction_counts[9];
    DensityClass density_states[9];
    ```
*   **Thresholds:** LOW (0–10 vehicles), MEDIUM (11–25 vehicles), HIGH (26+ vehicles).

### B. Ambulance Tracker (`ambulance_tracker.c`)
*   **Purpose:** Tracks ambulance positioning, velocity vectors, and active tracking variables as the vehicle progresses through the simulated city network.
*   **Inputs:** System trigger interrupt (button A0 press) and coordinates.
*   **Outputs:** Struct variables mapping the active journey state:
    ```c
    typedef struct {
        uint8_t current_node;        // Active junction ID (0 to 8)
        uint8_t target_hospital;     // Fixed at 8 (Junction I)
        float speed_mps;            // Speed in meters per second
        uint8_t distance_remaining; // Hops remaining to hospital
        uint8_t is_active;          // Run status (1 = running, 0 = idle)
    } AmbulanceState;
    ```

### C. Route Optimizer (`route_optimizer.c`)
*   **Purpose:** Computes the fastest path to Hospital I using a custom weighted Dijkstra routing search. 
*   **Routing Penalty Matrix:**
    *   Base road segment distance = 100 meters (nominal cost = 100).
    *   If target junction is `LOW_DENSITY`: cost multiplier = 1.0 (weight = 100).
    *   If target junction is `MED_DENSITY`: cost multiplier = 1.5 (weight = 150).
    *   If target junction is `HIGH_DENSITY`: cost multiplier = 3.0 (weight = 300).
*   **Outputs:** Path sequence array `uint8_t optimal_path[9]` (e.g. `[0, 3, 6, 7, 8]` for `A ➔ D ➔ G ➔ H ➔ I`).

### D. ETA Calculator (`eta_calculator.c`)
*   **Purpose:** Computes the remaining travel duration in seconds to the hospital, and lists predicted arrival intervals for each upcoming node on the route.
*   **Methodology:**
    $$\text{Segment Cost (sec)} = \frac{\text{Base Distance} \times \text{Congestion Multiplier}}{\text{Ambulance Speed}}$$
*   **Outputs:**
    *   `uint16_t global_eta_seconds` (time remaining to hospital).
    *   `uint16_t node_eta_seconds[9]` (ETA value specific to each future preemption target).

## 3. Communication Channel: Virtual SPI Master (`spi_master.c`)

The SPI Master module acts as the serialization interface. It runs the bitfield compiler and logs transaction records.

### A. Bit-Packing Logic
The packed SPI frame is defined as a union mapping a 32-bit register directly to the virtual protocol:

```c
typedef union {
    struct {
        uint8_t  checksum     : 8;  // Error check byte (XOR check)
        uint8_t  dist_remain  : 3;  // Distance remaining (0-7 nodes)
        uint16_t eta_seconds  : 12; // Time to target (0-4095s)
        uint8_t  target_node  : 4;  // Next junction to prepare (0-8)
        uint8_t  current_node : 4;  // Ambulance current position (0-8)
        uint8_t  emg_active   : 1;  // Emergency override assert
    } fields;
    uint32_t raw_data;              // 32-bit combined register word
} SpiFrame;
```

### B. Checksum Generation Algorithm
Before transmission, the checksum field is calculated via a byte-wise XOR operation of the top 3 bytes of the frame:
```c
uint8_t Calculate_XOR_Checksum(uint32_t raw_word) {
    uint8_t byte3 = (raw_word >> 24) & 0xFF;
    uint8_t byte2 = (raw_word >> 16) & 0xFF;
    uint8_t byte1 = (raw_word >> 8)  & 0xFF;
    return (byte3 ^ byte2 ^ byte1);
}
```

### C. Simulation Output Routine
For simulation verification, `spi_master.c` writes the transaction record as a 32-bit hexadecimal string to a shared virtual bus file:

```c
#include <stdio.h>

void SPI_Transmit_Frame(SpiFrame* frame) {
    frame->fields.checksum = Calculate_XOR_Checksum(frame->raw_data);
    
    // Virtual SPI transmission (Write to virtual register file)
    FILE *spi_bus = fopen("simulation/virtual_bus/spi_bus.txt", "w");
    if (spi_bus != NULL) {
        fprintf(spi_bus, "0x%08X\n", frame->raw_data);
        fclose(spi_bus);
    }
}
```

## 4. Verification and Mock Testing

To validate the firmware before integration, the `stm32/tests/` directory houses three unit testing harnesses:
1.  `test_route_optimizer.c` - Runs Dijkstra routes under varied congestion profiles (LOW, MED, HIGH traffic checks).
2.  `test_eta_calculator.c` - Verifies travel duration math and segment speed dividers.
3.  `test_interfaces.c` - Tests the bit-packer, checks struct alignment, and validates XOR checksum generation.
