# Digital Twin Implementation Guide
## Development, Simulation, and Execution Guide

---

## 1. Development Environment Setup

To run the simulation-first embedded digital twin, install the following development environments:

*   **STM32CubeIDE (v1.12.0 or higher):** Compiles and debugs the C-based traffic intelligence layer.
*   **Vivado Design Suite (v2020.2 or higher):** Synthesizes and simulates the Verilog HDL controller tier.
*   **Tinkercad Circuits (Web Browser):** Animates the Arduino LCD display and LED circuit boards.
*   **Python 3.9+:** Runs the integration orchestrator and performance graphs.

---

## 2. Directory Layout & Shared Assets

Before writing code, verify that the project structure matches the digital twin model:
*   Protocol definitions (SPI structures and UART mappings) are stored in `shared/protocols/`.
*   Virtual hardware buses (text-based register files) are located in `simulation/virtual_bus/`.
*   Test scenarios containing dynamic vehicle volumes are stored in `simulation/scenarios/`.

---

## 3. Subsystem Execution Instructions

### Step 1: Run the STM32 Traffic Intelligence Simulation
The STM32 firmware processes congestion inputs, determines routes using Dijkstra, and packages the results into a 32-bit SPI transaction:

1.  Open STM32CubeIDE and import the `stm32/` directory.
2.  Set the active compilation configuration to `Debug` or `Simulation`.
3.  Execute the unit test suite:
    ```bash
    gcc -I./include -I../shared/protocols ./tests/test_interfaces.c -o test_interfaces
    ./test_interfaces
    ```
4.  Run the main simulator program. It reads scenario data and writes the packed registers to the virtual SPI line:
    `simulation/virtual_bus/spi_bus.txt` (e.g. logging `0x808103C8`).

---

### Step 2: Run the Vivado FPGA RTL Simulation
The FPGA reads the virtual SPI log, validates the checksum, executes state transitions, and outputs serial UART states:

1.  Open Vivado and navigate to the project directory:
    ```tcl
    cd vivado/
    open_project Green_Corridor_Controller/Green_Corridor_Controller.xpr
    ```
2.  Add Verilog source files: `rtl/spi_slave_receiver.v`, `rtl/corridor_controller.v`, `rtl/emergency_fsm.v`, `rtl/signal_fsm.v`, `rtl/green_corridor_top.v`.
3.  Add the simulation testbench: `testbench/tb_green_corridor.v`.
4.  Run Behavioral Simulation:
    *   Click **Run Simulation** ➔ **Run Behavioral Simulation** in the Flow Navigator.
    *   The testbench reads `simulation/virtual_bus/spi_bus.txt` and drives SCLK/MOSI pins.
    *   The simulator compiles waveforms and writes the ASCII UART output stream to `simulation/virtual_bus/uart_bus.txt`.

---

### Step 3: Run the Tinkercad Arduino Display Driver
The Arduino acts as the physical UI, displaying signal lights and LCD characters:

1.  Open [Tinkercad Circuits](https://www.tinkercad.com/) in your web browser.
2.  Assemble the LED, push-button, and 16x2 LCD layout matching the schematic in `tinkercad/circuit_design/`.
3.  Open the Code panel, select **Text**, and copy the contents of `tinkercad/arduino_code/smart_city_traffic_controller.ino`.
4.  Click **Start Simulation**.
5.  Open the **Serial Monitor** at the bottom of the code panel.
6.  Paste the ASCII UART log string generated by the FPGA (e.g., `$2002D02\n`) into the input field and press Enter.
7.  Verify that LED A turns green and the LCD screen displays `AMB@D  ETA: 02:00`.

---

### Step 4: Closed-Loop Python Orchestration & Analytics
To automate this pipeline without manual copying:

1.  Navigate to the simulation directory:
    ```bash
    cd simulation/orchestrator/
    ```
2.  Install requirements:
    ```bash
    pip install matplotlib pandas
    ```
3.  Run the orchestrator script to bridge the logs:
    ```bash
    python run_digital_twin.py --scenario scenario_bypass.json
    ```
    *   This script feeds the scenario values to the STM32 binary, copies `spi_bus.txt` into the Vivado simulation directory, runs xsim in batch mode, extracts the resulting `uart_bus.txt`, and streams the output to the Python terminal dashboard.
4.  Run the comparative analytics dashboard:
    ```bash
    python plot_performance.py
    ```
    *   Generates graphs comparing travel durations with and without preemption, saving plots to the `results/` folder.
